THIS IS A PIXEL!

Hi, my name is Chuck.

Sometimes I like to think about pixels.

By themselves pixels are pretty boring

But in groups, they can represent amazing things!

Let's make the pixel to the left into a 4×4 grid of pixels, like this.

A 16 pixel image is pretty small, but this will still be fun!

For now let's restrict ourselves to monochrome images.

Each pixel can be in one of two states (black or white).

This gives us 2 states for 16 pixels, or 216 = 65,536 possible combinations.

In binary (using 16 bits) these numbers would look like this:

0 0000000000000000
1 0000000000000001
2 0000000000000010
3 0000000000000011
...
65533 1111111111111101
65534 1111111111111110
65535 1111111111111111

Don't worry that's as mathy as we'll get.

The point of all of this is, if we broke these 16 bit numbers into groups of 4 bits and stacked them on top of each we would have a bitmap.

Let's use zero (0000000000000000) as an example:

0000

← If zero represent white, and one represents black, then the 16 bit value for zero would be the image on the left.

What if we use one (0000000000000001) instead?

← We would get the new image on the left!

← You get the point.

Of course we could do this for every possible value, and generate every possible image.

What if we used a 24×24 instead of a 4×4 pixel image, and 256 colors instead of 2?

Things get out of hand quickly.

This number exceeds what your computer is capable of calculating without the help of special libraries designed to work with large numbers.
